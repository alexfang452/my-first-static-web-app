Team 2's Web Service API

Endpoints Method Output Description
/extensions GET Returns a Metadata file structured like this: { "version": "0.12", "components": [ { "component": { "id": "string", "uploader_id": "string", "name": "string", "description": "string", "versions": [], "flagged": "boolean", "flag_reason": "string" } } ] } Downloads the Extensions Metadata File for Viewing
/extensions/{id}/versions/{version_id} GET Abc123-v3-files.zip/ ,Metadata.txt ,Extension/ ,Dependencies/ Get a specific version of the extension files and its dependencies. Returned in a zip file like above.
/user/{user_id}/extensions POST { "ExtensionVersion": { "ExtensionID": "abc123", "NewVersion": { "VersionID": "v2.01", "Status": "Success", "Message": "New extension created successfully" } } } Allows User to submit a new extension to the Registry.
/user/{user_id}/extensions/{id}/version POST { "ExtensionVersion": { "ExtensionID": "abc123", "NewVersion": { "VersionID": "v2.01", "Status": "Success", "Message": "New version created successfully" } } } Allows User to submit a new version of an existing extension to the Registry.
/admin/extensions/{id}/version/{version_id}/flag POST 204 NO CONTENT Allows User to Flag an Extension for Review.
/admin/extensions/{id}/version/{version_id}/review POST 204 NO CONTENT Reviewers can review an Extension version, approve or deny it, and leave comments on it.
